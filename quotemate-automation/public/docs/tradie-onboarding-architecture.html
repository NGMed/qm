The Flow

Six steps, one button click at the end.

Each node below is a real page or process. Hitting "Activate my QuoteMate" on step 5 fires every backend integration in parallel.

01 Home Tradie lands on /, clicks "Get my QuoteMate".
02 Signup Page /signup. Business name, owner first name, email, password (4 fields).
03 Email verify Supabase sends a link. Tradie clicks → /auth/callback picks up session.
04 Wizard /onboard: trade + state + pricing essentials + review (3 steps).
05 Activate POST /api/onboard/activate runs the 7-step provisioning chain.
06 Live AI /onboard/success. Owner gets a welcome text from the new number.
The Activation Checklist

Seven things run when "Activate" is clicked.

Steps 1–3 are pure database writes (always real). Steps 4–7 hit external APIs and are gated by env flags so test phase costs nothing. Order is atomic — failure at any step rolls back or surfaces a clear warning.

01

Validate the payload

Zod schema in lib/onboard/schema.ts checks every field. Inline errors returned per-field for the wizard to render.

Always real
02

Insert tenants row

Owner identity, business name, mobile, trade, state. Status starts as 'onboarding'. Flipped to 'active' in step 6.

Supabase
03

Insert pricing_book + tenant_service_offerings

Hourly rate, markup, callout. Auto-enables the trade's easy-5 services. Defaults applied for any blank advanced field (apprentice rate, after-hours multiplier, etc.).

04

Provision AU Twilio number

AU only · Voice + SMS + MMS required, Fax best-effort. Sets SmsUrl → /api/sms/inbound, VoiceUrl → api.vapi.ai/twilio/inbound_call. No manual Twilio console clicks.

Database Schema

Three tables, one tenant_id across them.

Migration 015_tenants_onboarding.sql is the foundation. Every operational table (intakes, quotes, calls, sms_conversations, customers) also gained a tenant_id column for downstream scoping.

tenants

One row per registered tradie. Owns pricing, branding, and the provisioned Twilio + Vapi identifiers.

create table tenants (
  id                   uuid primary key,
  owner_user_id        uuid -> auth.users,
  business_name        text not null,
  owner_first_name     text,
  owner_email          text unique,
  owner_mobile         text not null,
  trade                text -- electrical | plumbing
  state                text,
  abn                  text,
  licence_type         text,
  licence_number       text,
  licence_expiry       date,
  twilio_sms_number    text unique,
  twilio_voice_number  text unique,
  vapi_assistant_id    text,
  vapi_voice_persona   text,
  stripe_connect_account_id text,
  status               text -- onboarding | active
  created_at           timestamptz,
  activated_at         timestamptz
);

pricing_book (extended)

Pre-existing table. Migration 015 added tenant_id, senior_rate, after_hours_multiplier. One row per (tenant, trade).

alter table pricing_book
  add tenant_id              uuid -> tenants(id),
  add senior_rate            numeric(8,2),
  add after_hours_multiplier numeric(4,2)
        default 1.5;

-- Existing columns that matter:
--   hourly_rate, call_out_minimum,
--   apprentice_rate, default_markup_pct,
--   risk_buffer_pct, min_labour_hours,
--   gst_registered,
--   licence_type/number/state/expiry

create unique index
  pricing_book_tenant_trade_unique
  on pricing_book (tenant_id, trade)
  where tenant_id is not null;

tenant_service_offerings

Which catalogue items the tradie offers. Auto-populated with the easy-5 for their trade on activate.

create table tenant_service_offerings (
  tenant_id    uuid -> tenants(id),
  assembly_id  uuid -> shared_assemblies(id),
  enabled      boolean default true,
  primary key (tenant_id, assembly_id)
);

tenant_id added to operational tables

Every downstream table got the tenant_id column so quote drafting, intake structuring, and SMS dialogs can scope per-tradie. NULL allowed for back-compat with pre-v6 conversations.

alter table intakes
  add tenant_id uuid -> tenants(id);

alter table quotes
  add tenant_id uuid -> tenants(id);

alter table calls
  add tenant_id uuid -> tenants(id);

alter table sms_conversations
  add tenant_id uuid -> tenants(id);

alter table customers
  add tenant_id uuid -> tenants(id);
External Services

Three integrations, zero manual config.

Each tradie gets their own resources in all three external services. The activate endpoint orchestrates the creation; the tradie never opens any of these dashboards themselves.

Supabase
Auth (sign up + email verify + sign in). Postgres for tenants, pricing_book, intakes, quotes. RLS planned for v6.x.
Env:
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY
SUPABASE_SERVICE_ROLE_KEY
SUPABASE_DB_URL
Twilio
Buys the tenant's AU phone number. Pre-configures SMS webhook to our app, voice webhook to Vapi's hosted endpoint. Sends the welcome SMS from the new number.
Env:
TWILIO_ACCOUNT_SID
TWILIO_AUTH_TOKEN
APP_URL (webhooks)
TWILIO_PROVISIONING_ENABLED
Vapi
Creates per-tenant AI assistant (Claude + 11labs + Deepgram). Registers the Twilio number against the assistant so inbound calls run THAT specific assistant.
Env:
VAPI_API_KEY
VAPI_VOICE_JON (optional)
VAPI_VOICE_SARAH (optional)
VAPI_PROVISIONING_ENABLED
Webhook Auto-config

Every new Twilio number is born wired up.

Set at purchase time by the Twilio API call. Tradies never open the Twilio console; numbers arrive with both webhooks pre-configured.

SMS webhook

Customer SMS lands on our app. The inbound route resolves tenant by destination number, then runs the dialog Haiku with the right pricing book.

URL:    https://quote-mate-rho.vercel.app
       /api/sms/inbound
METHOD: POST

Voice webhook

Customer call lands on Vapi's hosted endpoint. Vapi looks up the destination number in its phone-number table (registered in step 5b) and runs the bound assistant.

URL:    https://api.vapi.ai
       /twilio/inbound_call
METHOD: POST
Activation Flags

Two flags decide stub vs real.

Test phase runs with both flags off — no Twilio money, no Vapi calls, but every UI screen + database write works end to end. Flip to true (per environment) when you're ready to provision real numbers and assistants.

Environment toggles

TWILIO_PROVISIONING_ENABLED=true → real number buy + welcome SMS
VAPI_PROVISIONING_ENABLED=true → real assistant create + number register
APP_URL=https://quote-mate-rho.vercel.app → webhook destination
