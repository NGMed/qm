The Receptionist · Pricing flow

HOW A TEXT BECOMES A PRICED QUOTE

A customer asks for a quote. Sixty seconds later they get three options — Good, Better, Best — with real numbers and a deposit link.

This page shows the data shapes that move between each engine, the database rows that get pulled, and what the final quote looks like in storage.

The pipeline

FOUR STOPS, START TO FINISH.

01

Customer message

A text or call. The raw request — messy, partial, conversational.

02

Intake Engine

Reads the message. Turns it into a tidy JSON record: what, how many, where, who.

03

Estimation Engine

Looks up the tradie's rates + catalogue. Builds three priced tiers.

04

Safety check

Every $ has to trace back to a real DB row, or the quote becomes a $99 site visit.

The full flow

FROM MESSAGE TO PRICED QUOTE.

Pipeline · data flows left to right

Each engine hands a tidier, more structured packet to the next. The database feeds both the Estimator and the Safety check.

Where prices come from

TWO DATABASES. NO GUESSING.

Every dollar comes from one of two database tables. Both are set by the tradie during onboarding.

Source A · Your rates

The Pricing Book

One row per tradie. Set once. Holds the numbers that apply to every job.

  • Hourly rate — e.g. $118/hr
  • Call-out minimum — e.g. $160
  • Markup % on materials — e.g. 36%
  • Apprentice + senior rates
  • After-hours multiplier — e.g. ×2.0
  • Min labour hours per job
  • GST registered?
Source B · Your products + services

The Catalogue

The actual things you install. Shared library + your customisations.

  • Assemblies — "install a downlight" = 0.5hr + material
  • Materials — real branded products
  • Your custom items — services only you offer
  • Customer-supply price — install-only rate
  • Tier hints — Good / Better / Best

02 · Intake Engine

THE LISTENER.

Turns a messy human message into structured JSON the Estimator can rely on.

What comes in

  • SMS text or voice transcript.
  • Photos the customer uploaded.
  • Phone number, name (if known).

What the AI is told

  • Pull out: what, how many, where, who, when.
  • Tag the trade from the job type.
  • Capture specs (warm-white, dimmable, brand).
  • Mark if the customer is supplying the product.
  • Decide: safe to quote, or needs inspection?

What comes out

A tidy JSON record the Estimator can read.

See the data shapes below.

03 · Estimation Engine

THE PRICER.

Takes the structured intake and builds a Good / Better / Best quote using the tradie's real Pricing Book and Catalogue.

What comes in

  • Structured intake JSON.
  • The tradie's Pricing Book row.
  • Their Catalogue (assemblies + materials).

What the AI is told

  • Use only prices from the tool calls — never invent.
  • Build three tiers, different product brand per tier.
  • Top labour to the min job allowance (e.g. 3 hrs).
  • Use customer-supply pricing when the customer brings their own.
  • Fall back to a $99 inspection rather than guess.

What comes out

Three priced tiers, each with line items and a subtotal — handed to the Safety check.

04 · Safety check

THE PROVER.

Walks every line item. Each price must trace back to a real DB row, or the whole quote becomes a $99 site visit.

What comes in

  • The draft quote (Opus output).
  • The Pricing Book row.
  • Every candidate price built from shared + tenant catalogues.

What the checker does

  • Labour lines → match hourly / apprentice / senior rate.
  • After-hours lines → match hourly × multiplier.
  • Call-out lines → match call_out_minimum.
  • Material lines → match a real DB row (±$0.50, ±5pp markup).
  • Category check: "downlight" line can't price from a "GPO" row.
  • Labour-floor check: tier must bill ≥ min_labour_hours.

What comes out

Pass → quote ships as Good / Better / Best with deposit links.

Fail → whole quote becomes a $99 site-visit link.

▼ How one line item gets checked
Validator decision tree · runs once per line
▼ PASS — every line grounded 
{
  "valid": true
}
▼ FAIL — at least one line can't be traced back 
{
  "valid": false,
  "failures": [
    {
      "tier": "better",
      "lineIndex": 2,
      "description": "Supervisor fee",
      "unit": "each",
      "unit_price_ex_gst": 400,
      "expected": "shared_materials/shared_assemblies (raw or × 36% markup)"
    }
  ]
}

On FAIL, the route handler wipes all three tiers to null, sets needs_inspection: true, and ships only the $99 site-visit link.

Why the price is trustworthy

THREE GUARDRAILS ON EVERY QUOTE.

AI invents things. The Estimator assumes it will, and refuses to let invented prices through.

Tool-calling only

Opus cannot type a dollar amount into a line item. The only way to get a price is to ask the database for it via a tool call.

Grounding validator

A non-AI checker reviews every line. Any price that can't be traced → whole quote becomes a $99 inspection.

$99 fallback

When anything is uncertain — missing customer-supply price, unfamiliar job — the system books a paid site visit. Fail safe.

A real example, end to end

SIX DOWNLIGHTS IN SARAH'S LOUNGE.

  1. 01 · Customer. Sarah texts: "hi can i get a quote for 6 downlights in my lounge, dimmable warm-white, north sydney"
  2. 02 · Intake Engine. Reads the text. Writes a row to intakes with job_type: "downlights", count: 6, specs: { color_temp, dimmable: true }, trade: "electrical".
  3. 03 · Estimation Engine. Loads Sarah's sparky's Pricing Book row + Catalogue. Opus calls lookup_material 3 times (one per tier), each time the tool returns the top 5 matching products from tenant_material_catalogue. Opus assembles three priced tiers using the matching assembly ("Install LED downlight", default_labour_hours: 0.4) and the chosen product per tier.
    Call-out fee$160.00
    6 × LED downlights ($32 × 1.36 markup = $43.52 each)$261.12
    Labour: 6 × 0.4 hr = 2.4 hr, floored to min_labour_hours = 3 hr × $118/hr$354.00
    Subtotal ex GST · "Better" tier$775.12
    Customer sees (× 1.10 GST)$852.63
  4. 04 · Safety check. Walks every line of every tier. $160 = call_out_minimum ✓ · $43.52 = $32 × 1.36 markup ✓ · $118 = hourly_rate ✓ · 3 hrs ≥ min labour ✓ · "downlight" line tag matches "downlight" candidate tag ✓. Pass.
  5. 05 · Quote row written. All three tiers + Stripe links persisted to quotes. routing_decision is 'tradie_review' by default in v1, or 'auto_send' when V3_AUTOSEND_ENABLED=1 and intake confidence is HIGH (it is here). SMS dispatched to Sarah.
  6. 06 · Sarah sees. Three tiers — Good $762.87, Better $852.63, Best $1,211.67 (all inc GST). One tap on a tier locks it in with a 30% Stripe deposit.
